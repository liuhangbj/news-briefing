# 新闻简报生成指南（深度版）

> 版本：v1.0  
> 适用范围：每日新闻简报深度版（由 AI 手工操作，不依赖脚本）

---

## 一、操作原则

1. **全部手工操作，不依赖脚本**
   - 逐条阅读标题和摘要，逐条判断评级、分类、归属
   - 禁止使用批量分类/批量评分脚本替代人工判断
   - **必须逐条阅读候选列表中的全部文章，不得跳过、不得提前终止**

2. **逐条是底线，必须完成全部**
   - 每篇文章必须独立阅读标题 + 摘要，判断：国内/国际、板块、话题、事件、评级
   - 不得因"看起来是同一类"而批量处理
   - **不得因批次、时间或数量原因中途停止；必须完成全部文章后再进入下一步**

3. **信息不足时不猜测，标记待确认**
   - 如果标题和摘要不足以判断话题或事件，标记为"待确认"而非强行归类
   - 综述写作时，如果核心数据缺失（如被 Cloudflare 拦截导致无全文），在综述中标注数据来源限制，不编造数据

4. **分级有标准，不凭感觉**
   - S 级：利率政策、主要经济数据、大宗商品价格、地缘冲突升级
   - A 级：能源行业、钢铁/矿业、贸易政策、中国宏观、国际组织、跨境融资
   - B 级：科技/AI、航天、资本市场、企业变动、世界杯、强信号词、汇率波动（仅当波动显著或关联重大政策时）
   - C 级：跳过（体育、娱乐、社会新闻、纯表态），但 C 中若有热点潜质（如重大社会事件、突发事故）可保留备选

5. **事件级别聚类，不是分类级别**
   - 事件 = 有明确时间锚点（具体日期）+ 主体（谁）+ 结果（发生了什么）
   - 分类 = 泛泛主题类别（如"中国贸易""并购市场"），无具体事件锚点，不能作为热点话题
   - 例如"非农就业报告"是事件（2026年6月5日发布，新增17.2万），"芯片行业"是分类不是事件

## 二、九步操作流程

### 第 1 步：逐条阅读与评级（必须完成全部）

**必须逐条阅读候选列表中的每一篇文章，不能跳过、不能分批终止、不能因数量大而偷懒。**

对每一篇文章阅读标题 + 摘要，判断：
- **国内/国际**：是否涉及中国或中国机构/企业/政策？是 → 国内；否则 → 国际
- **板块**：政治、宏观、企业、社会（国内/国际）；基础科学、AI、其他（科技）；文化、体育、娱乐、生活方式（文体）
- **话题与事件**：这篇文章属于哪个具体事件？（如"非农就业报告""美伊冲突升级"）还是泛泛分类？
- **评级**：S / A / B / C（按标准判断）

**输出**：一篇文章对应一条记录，格式：
```
[评级] 标题 | 国内/国际/板块 | 事件/分类 | 关键词
```

### 第 2 步：入选与备选

- S、A、B 直接入选
- C 类逐条审视：是否有热点潜质？（突发事故、重大社会事件、政策变动等）→ 保留备选；否则剔除
- 世界杯不逐条保留，汇总成话题（见第 3 步）

### 第 3 步：事件聚类与热点话题筛选

1. 从入选文章中提取所有事件（按第 1 步记录的事件标签）
2. 统计每个事件的文章数量
3. 文章数量 ≥ 3 且事件级别（有时间锚点）→ 列为候选热点话题
4. 从候选热点话题中筛选最终热点：
   - 优先 S 级事件密集的话题
   - 优先有核心数据支撑的话题（如出口从190万桶→20.9万桶）
   - 优先有明确市场/政策影响的话题（如非农就业→加息预期）
5. 世界杯：不单独成热点话题，但如果当日有大量相关文章且属于 S/A 级讨论（如经济影响、地缘政治），可纳入

### 第 4 步：逐条检查热点话题内文章匹配度

对进入热点话题的每一篇文章，检查：
- 标题+摘要是否确实属于该事件？
- 是否只是关键词匹配但实质内容不同？（如"大跌之夜高盛喊抄底"被误分到"美伊冲突"）
- 是否应拆分为独立事件？（如"SpaceX IPO"和"SpaceX与谷歌算力协议"是两个独立事件）

**不匹配的文章移出热点话题，回到第 7 步的分类列表**

### 第 5 步：逐条检查非热点文章是否遗漏进热点

对未进入热点话题的文章，逐条检查：
- 是否属于某个热点话题但第 1 步被错误标记？（如"特朗普关税"文章被标记为"美伊冲突"）
- 是否属于某个热点话题但事件标签不同？（如"芯片暴跌"和"美股黑色星期五"实质是同一事件）
- 是否应新增为热点话题？（如某个事件只有 2 篇文章但 S 级）

### 第 6 步：撰写热点话题综述（≤600字）

对每个热点话题：
1. 读取该话题下所有文章的缓存全文（或可用摘要）
2. 如果 Bloomberg/Reuters 等文章被 Cloudflare 拦截（显示 "We've detected unusual activity"），**不爬取网页补全文**，仅在综述中标注"部分来源全文受限，基于中文源完整描述 + 标题推断"
3. 综述结构：
   - **首段**（100-150字）：事件概述 — 时间、主体、核心结果
   - **中段**（200-300字）：关键数据/政策细节/影响链条 — 逐条列出核心数据点
   - **尾段**（100-150字）：市场/政策影响判断 — 对后续走势或决策的指向
4. 每个热点话题推荐 2-3 篇必读文章，格式：
   ```
   [华尔街日报]《封锁、制裁双重挤压，伊朗原油出口降至六年低点》(超链接)
       核心数据：出口从190万桶/日→20.9万桶/日，两个月骤降逾九成
   ```
   推荐文章用 `[**《标题》**](URL)` 格式嵌入完整 URL

### 第 7 步：非热点文章分类罗列

按以下分类体系：

**国内**：政治、宏观、企业、社会  
**国际**：政治、宏观、企业、社会  
**科技**：基础科学、AI、其他  
**文体**：文化、体育、娱乐、生活方式

罗列格式：
```
政治：
[路透社] 消息人士称，随着人民币走强，中国银行上调美元存款利率（超链接） 重要程度：S
```

### 第 8 步：逐条检查分类正确性

对第 7 步的分类结果逐条检查：
- 国内企业是否被分到国际/企业？（如华为、新华网）
- 国际事件是否被分到国内？（如欧盟贸易政策）
- 科技文章是否被分到文体？（如 AI 相关）
- 文体文章是否被分到科技？（如娱乐新闻）
- 政治/宏观/社会边界是否正确？（如台湾通胀是政治还是宏观？）

**所有检查必须在分类完成后手动做，不能跳过**

### 第 9 步：模板组装与输出

按以下模板组装最终文件，保存为 Markdown 格式。

### 格式规范（必须严格遵守）

1. **综述部分**：用纯段落（无项目符号），段落间空一行，至少 2-3 段
2. **相关阅读**：`1. [标题](URL)（来源）` 独占一行，后面紧跟 `- 核心信息：...` 子项
3. **板块标题**：用 `#### 板块名`（四个#），板块间空一行
4. **快读列表**：`- [标题](URL)（来源） — 核心信息` 一行一条，不换行
5. **严禁**：HTML 标签、`- ` 项目符号用于综述、省略空行

## 四、简报生成后处理流程（已自动化）

1. **保存为 Markdown**：按上述模板组装完整 Markdown 内容
2. **调用 `save_final_briefing()`**：脚本自动检测格式，调用 `md2html.py` 转换为 HTML
3. **自动推送**：GitHub Pages + Cubox 双通道推送

AI 只需完成第 1 步（生成 Markdown），后续全部自动化。
>
> **转换说明**：
> - 综述正文（`**综述**：` 下方段落）→ 自动合并为一个 `<blockquote>` 块引用（浅灰底色）
> - 相关阅读列表（`1. [标题](URL)`）→ `<ol>` + `<li>` 有序列表
> - 其他快读列表（`- 条目`）→ `<ul>` + `<li>` 无序列表
> - 标题 → 带颜色渲染的 `<h1>`~`<h4>`
> - 所有内链格式 → 完整 `<a href>` 超链接
> - AI 只需保证 Markdown 格式正确，HTML 渲染由脚本处理

## 四、常见错误清单（实操备忘）

| 错误类型 | 示例 | 修正方法 |
|---------|------|---------|
| **漏读文章** | 只读了190篇，漏掉656篇未读 | 必须逐条完成全部，不因批次或数量大而中途停止 |
| 国内/国际错位 | 华为云、新华网被分到国际/企业 | 以"中国企业/机构/政策"为首要判断标准，不看报道来源 |
| 事件拆分错误 | SpaceX IPO 和 SpaceX算力协议合并 | 检查时间、主体、结果是否不同，不同则拆分 |
| 事件合并错误 | 芯片暴跌和美股黑色星期五合并 | 检查是否为同一事件的不同报道，不同事件不合并 |
| 关键词误触发 | "大跌"同时出现在芯片暴跌和美股下跌中 | 以主体+结果为判断依据，而非关键词 |
| C级误入选 | 纯表态、无实质内容的官员发言 | 剔除；除非表态本身引发重大市场波动 |
| 综述数据编造 | Bloomberg全文缺失时编造数字 | 标注数据来源限制，不写未知数据 |
| 推荐文章无链接 | 仅有标题和来源 | 必须嵌入完整URL，用 `[**《标题》**](URL)` 格式 |

## 五、效率与质量要求

- 文章数量 100-200 篇时，预计耗时 30-45 分钟（逐条阅读 + 评级 + 聚类）
- 文章数量 200-500 篇时，预计耗时 60-90 分钟
- **无论文章数量多少，质量不可妥协**
- 分类边界有模糊地带（如台湾通胀算政治还是宏观）时，记录判断依据，后续统一修正

## 六、格式规范速查（输出模板标准）

简报正文必须遵循以下格式，避免每次重新调整：

| 元素 | 规范 |
|------|------|
| **来源栏** | 只列具体媒体名称，不列聚合器（如 Inoreader / FreshRSS） |
| **数据栏** | 简化格式：`24小时窗口内 NNN 条，筛选后基于 MMM 条新闻生成` |
| **信息限制标注** | **删除** — 不在正文中出现 Cloudflare 拦截说明 |
| **综述标题** | `**综述**：` — 不附加 `(≤600字)` 等字数提示 |
| **相关阅读标题** | `[**标题**](URL)` — 不用 `《》` 书名号 |
| **其他快读条目** | `[来源] 标题（超链接）` — 去掉 `[S/A/B/C]` 前缀、去掉 `《》`、去掉"重要程度：X" |
| **空板块** | 完全省略 — 没有内容的板块（文化、生活方式、五·其他）不保留标题 |
| **底部** | 只保留 `*简报生成时间：...*`，删除筛选逻辑和 AI 操作声明 |
